02. Pain Points

The Cloud Surveillance Bottlenecks

Slide 02
🚨 1. Passive Monitoring & Guard Fatigue

Traditional setups are forensic, not proactive:

  • Guards watching dozens of CCTV screens experience a 95% drop in attention span within 20 minutes, leading to missed real-time anomalies.
  • Attempts to route feeds to Cloud AI to automate monitoring fail due to latency and bandwidth constraints.
💻 2. Bandwidth & Latency Trap

Streaming raw video to the cloud is unsustainable:

  • Streaming 24/7 high-definition raw feeds from dozens of cameras exhausts network uplinks and incurs massive internet hosting bills.
  • Network latency (~200ms) prevents instantaneous alerts needed for active safety hazards.
⚖️ 3. Privacy & Legal Violations (DPDP)

Data privacy laws make centralized storage a liability:

  • Under India's new Digital Personal Data Protection (DPDP) Act, 2023, streaming and storing public surveillance footage centrally on external servers invites strict penalties (fines up to ₹250 Crore).
💰 4. High Infrastructure Capital (CapEx)

Upgrading hardware is financially restrictive:

  • Tearing down existing "dumb" analog or IP camera networks to install proprietary "AI Smart Cameras" is extremely expensive and economically non-viable for most institutions.

03. Solution

Proposed Solution: Edge AI + Federated Learning

Slide 03
⚡ What FedVision Does
  • Edge Inference: Object detection (Faster R-CNN, 41M params) runs locally on a retrofit edge gateway box connected to existing cameras.
  • Active Learning: Automatically selects key training frames (confidence: 50–75%) to minimize local storage.
  • Privacy-Preserving: Video stays local and is permanently deleted after processing. Only mathematical weight updates (~10MB) are shared over 5G.
  • Flower Aggregation: Central server runs FedAvg to average weights and construct a "Super Model".
  • Continuous Smart Updates: The global super-model is pushed back to edge cameras, making the entire network instantly smarter.

04. Tech Execution

Technical Execution & Validation Metrics

Slide 04
🛠️ Developed Codebase & Pipeline

We have built a fully functional end-to-end Python prototype containing:

  • AI Engine: PyTorch Faster R-CNN (ResNet-50 backbone with FPN neck, 41M parameters). We freeze backbone parameters for fast edge fine-tuning.
  • Federated Sync: Flower framework (gRPC, SSL disabled on localhost, ready for TLS) with custom SaveModelStrategy to preserve global model state across runs.
  • Local Data Generator: Self-supervised frame extraction with pseudo-labeling using high confidence threshold (threshold >= 0.8) to prevent label noise.
  • Double-Buffered Hot Swapping: Dual-thread execution allows live 24/7 inference in Thread 1 while training in Thread 2. Hot-swaps the model in <50ms with zero surveillance downtime.
